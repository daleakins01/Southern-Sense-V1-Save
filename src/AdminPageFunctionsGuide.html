Southern Sense Admin Panel User Guide

This panel gives you full control over the e-commerce operations, product catalog, and public-facing content. All changes are saved instantly to the database.

1. Orders Tab (Order Management)

Purpose: Track all sales and monitor shipping status.

  • Data Source: Reads all documents from the /orders collection.
  • Functionality: Lists orders, sorted newest first. Displays Order # (truncated), Date, Total, Status (Paid, Pending, etc.), and Customer (Guest or Authenticated).
  • Action: The "Details" button provides a popup with full customer shipping information and itemized line items.

2. Products Tab (Catalog & Inventory)

Purpose: Manage inventory, pricing, and product visibility.

  • Data Source: Reads and writes to the /products collection.
  • Functionality: Lists all products with their Scent Family, Price, Stock Level, and **Featured** status.
  • Action: The "Edit" button opens a modal allowing instantaneous updates to `name`, `price`, `stock`, `scentFamily`, `category`, and the critical `featured: boolean` flag (used on the Homepage).

3. Reviews Tab (Social Proof Moderation)

Purpose: Moderate and approve customer feedback before it appears on product pages.

  • Data Source: Reads and writes to the /reviews collection.
  • Functionality: Fetches all reviews, prioritizing those with approved: false (Pending).
  • Actions:
    • **Approve:** Sets `approved: true`. Review immediately appears on the corresponding product page.
    • **Unapprove:** Sets `approved: false`. Hides the review.
    • **Delete Permanently:** Removes the review from the database entirely.

4. Stock Alerts Tab (Pillar 3 Management)

Purpose: View and clear back-in-stock requests from customers.

  • Data Source: Reads and deletes from the /backInStockRequests collection.
  • Functionality: Lists all unresolved requests (those still in the database).
  • Actions:
    • **Email Customer:** Provides a quick `mailto:` link to notify the customer of restock.
    • **Mark as Resolved:** Deletes the request from the database, clearing the alert from the Admin queue.

5. Site Content Tab (CMS)

Purpose: Update static content pages without touching code.

  • Data Source: Reads and writes to the /siteContent collection (documents: about, candleCare, refillProgram).
  • Functionality: Dynamically loads all text fields from the selected document and allows the Admin to edit and save changes instantly. Public pages automatically reflect these updates.
